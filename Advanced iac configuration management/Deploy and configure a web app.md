# Deploy and configure a web app (Terraform + Ansible)

## **Objective:**

Provision an EC2 instance with Terraform and configure a simple Nginx web server with Ansible using SSH key-based authentication generated by Terraform.

---

## **Instructions**

### **Terraform provisioning:**

- Define a Security Group allowing HTTP (port 80) and SSH (port 22)

- Define a Key Pair

- Define an EC2 t2.micro instance running Amazon Linux 2

### **Outputs:**

- Public IP and DNS

- SSH user

- Key name and path

### **SSH key management:**

- Generate a local key pair

- Register the public key as an AWS key pair

- Ensure Ansible uses the private key for authentication

### **Ansible configuration:**

- inventory.ini with the EC2 public IP

- site.yml to:

  - Install Nginx

  - Enable and start the service

  - Deploy the web application

  - Use a handler for service restarts

### **Verification:**

- Run curl against the EC2 public IP

- Confirm the app is accessible in a browser

- Capture a browser screenshot as evidence

### **Cleanup:**

- Run terraform destroy after verification

## **Tips**

- Generate your SSH key pair before running Terraform so the public key is ready to register

- Store the private key securely and never commit it to your GitHub repository — add it to .gitignore

- Use ansible_user=ec2-user in your inventory.ini for Amazon Linux 2

- Add StrictHostKeyChecking=no to your Ansible SSH options to avoid connection prompts on first run

- Test your SSH connection manually before running the Ansible playbook to confirm key-based authentication works

- Tag all Terraform resources for easy identification during verification and cleanup
